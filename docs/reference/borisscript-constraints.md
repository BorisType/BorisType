# Ограничения языка BorisScript

## Обзор

BorisScript (BS) — это **JavaScript-подобный язык сценариев** для платформы WebSoft HCM со **значительными ограничениями**. Он синтаксически похож на JavaScript, но **не является JavaScript** и выполняется в ограниченном окружении.

**Цель BorisType:** Обеспечить более выразительный код путём транспиляции TypeScript в валидный BorisScript.

---

## Переменные и области видимости

### Поддерживается только `var`

| Возможность               | JavaScript     | BorisScript                |
| ------------------------- | -------------- | -------------------------- |
| `var`                     | ✅             | ✅                         |
| `let`                     | ✅             | ❌                         |
| `const`                   | ✅             | ❌                         |
| Блочная область видимости | ✅ (let/const) | ❌ (только функциональная) |

**Следствие:** BorisType транспилирует `let`/`const` в `var`.

### Запрещено повторное объявление в одной области видимости

```javascript
// ❌ Ошибка в BorisScript
function test() {
  var x = 1;
  var x = 2; // Ошибка повторного объявления!
}

// ✅ Правильно
function test() {
  var x;
  x = 1;
  x = 2; // Присваивание OK
}
```

**Следствие:** BorisType поднимает все объявления переменных в начало функции/модуля.

### Нет захвата переменных между контекстами

```javascript
// ❌ Ошибка в BorisScript
var callbacks = [];
for (var i = 0; i < 3; i++) {
  callbacks.push(function () {
    alert(i); // Ошибка: i не определена в этом контексте
  });
}

// ✅ Правильно (требуется механизм __env)
// BorisType генерирует __env цепочки для замыканий
```

**Следствие:** BorisType использует объекты `__env` для захваченных переменных.

## Управление потоком

### Циклы

| Тип цикла            | JavaScript | BorisScript            |
| -------------------- | ---------- | ---------------------- |
| `for` (классический) | ✅         | ✅                     |
| `for...in`           | ✅         | ✅ (другое поведение!) |
| `for...of`           | ✅         | ❌                     |
| `while`              | ✅         | ✅                     |
| `do...while`         | ✅         | ✅                     |

### Различие в поведении for...in

**JavaScript:** Итерирует по **ключам** (индексам для массивов)

```javascript
const arr = [10, 20, 30];
for (let key in arr) {
  console.log(key); // "0", "1", "2" (indices)
}
```

**BorisScript:** Итерирует по **значениям** для массивов

```javascript
var arr = [10, 20, 30];
for (var item in arr) {
  alert(item); // 10, 20, 30 (значения!)
}
```

**Следствие:** BorisType транспилирует `for...of` → `for...in` (работает корректно).

## Функции

### Нет стрелочных функций

```javascript
// ❌ Не поддерживается
const multiply = (x, y) => x * y;

// ✅ Используйте function-объявления/выражения
function multiply(x, y) {
  return x * y;
}
```

**Следствие:** BorisType транспилирует стрелочные функции в обычные.

### Нет async/await

```javascript
// ❌ Не поддерживается
async function fetchData() {
  const result = await fetch(url);
}

// ✅ Используйте callback'и или синхронный код
```

## Объекты и массивы

### Объекты — это простые словари

```javascript
// ✅ Простые объекты работают
var obj = { name: "John", age: 30 };

// ❌ Нет прототипов или классов
function Person(name) {
  this.name = name;
}
var person = new Person("John"); // Не поддерживается
```

### Ключи объектов — только строки

```javascript
// JavaScript: любой тип может быть ключом
const map = new Map();
map.set({ id: 1 }, "value");

// BorisScript: только строки
var obj = {};
obj["key1"] = "value"; // ✅ OK
obj[123] = "value"; // Преобразуется в "123"
obj[{ id: 1 }] = "x"; // ❌ toString() → "[object Object]" (бесполезно)
```

**Следствие:** Избегайте использования объектов как Map с нестроковыми ключами.

## Современные возможности JavaScript

### Не поддерживаются

| Возможность                  | Статус                            |
| ---------------------------- | --------------------------------- |
| **Итераторы**                | ❌                                |
| **Генераторы** (`function*`) | ❌                                |
| **Promises**                 | ❌                                |
| **async/await**              | ❌                                |
| **Классы** (`class`)         | ❌                                |
| **Наследование** (прототипы) | ❌                                |
| **Symbols**                  | ❌                                |
| **WeakMap/WeakSet**          | ❌                                |
| **Proxy/Reflect**            | ❌                                |
| **Шаблонные литералы**       | ❌ (транспилируется в `+`)        |
| **Деструктуризация**         | ❌ (транспилируется)              |
| **Spread оператор** (`...`)  | ❌ (транспилируется)              |
| **Rest параметры**           | ❌ (транспилируется)              |
| **Параметры по умолчанию**   | ✅ (частично, через транспиляцию) |

## Операторы

### Приоритет операторов НАРУШЕН

**JavaScript:** Стандартный приоритет (`*` перед `+`)

```javascript
2 + 3 * 4; // 14 (3*4=12, 12+2=14)
```

**BorisScript:** **Вычисление слева направо** (НЕПРАВИЛЬНО!)

```javascript
2 + 3 * 4; // 20 (2+3=5, 5*4=20) ⚠️
```

**Следствие:** Эмиттер BorisType добавляет скобки для обеспечения правильного порядка:

```javascript
2 + 3 * 4; // Принудительно умножение первым
```

### Ternary/Conditional внутри бинарных выражений

Из-за left-to-right парсинга, **conditional expressions** (ternary `? :`) внутри бинарных операторов парсятся некорректно:

```javascript
// ❌ BS парсит неправильно — + уходит внутрь ternary
"PREFIX " +
  (cond ? a : b) +
  " SUFFIX"(
    // Парсится как: "PREFIX " + cond ? a : (b + " SUFFIX")

    // ❌ BS парсит неправильно — === уходит в alternate
    cond ? a : b,
  ) ===
  42;
// Парсится как: cond ? a : (b === 42)
```

**Следствие:** BorisType извлекает conditional expressions во временные переменные (Expression Extraction):

```javascript
// Исходный TypeScript:
const result = `PREFIX ${obj?.a?.b?.c} SUFFIX`;

// Сгенерированный BorisScript:
var __tmp0;
var __oc0;
__oc0 = (__tmp0 = ...) == null || __tmp0 == undefined ? undefined : bt.getProperty(__tmp0, "c");
result = "PREFIX " + __oc0 + " SUFFIX";
```

### Имена переменных

```javascript
// ❌ Не разрешено
var $price = 100; // $ не разрешён в именах

// ✅ Разрешено
var price = 100;
var _price = 100; // подчеркивание OK
```

## Синтаксические ограничения

### Скобки вокруг объекта при доступе к свойству

```javascript
// ❌ Синтаксическая ошибка в BorisScript
obj.key;

// ✅ Правильно
obj.key;

// ✅ Если необходимо, используйте переменную
var temp = obj;
temp.key;
```

---

## Система типов

### Динамическая типизация (как в JavaScript)

```javascript
var value = 42; // number
value = "hello"; // теперь string (OK)
```

### Нет проверки типов во время выполнения

```javascript
var arr = [1, 2, 3];
arr.nonExistent(); // Runtime ошибка (не graceful проверка на undefined)
```

**Следствие:** BorisType использует обертку `bt.getProperty()` для graceful обработки отсутствующих свойств.

## Что работает

Несмотря на ограничения, BorisScript поддерживает:

| Возможность            | Детали                                                           |
| ---------------------- | ---------------------------------------------------------------- |
| **Переменные**         | `var` с функциональной областью видимости                        |
| **Функции**            | Обычные функции, замыкания (через `__env`)                       |
| **Объекты**            | Простые словари `{ key: value }`                                 |
| **Массивы**            | `[1, 2, 3]` с базовыми методами                                  |
| **Условия**            | `if`, `else`, `switch`                                           |
| **Циклы**              | `for`, `for...in`, `while`, `do...while`                         |
| **Операторы**          | Арифметические, сравнения, логические (с оговоркой о приоритете) |
| **Методы строк**       | Базовые: `split`, `substring`, `toLowerCase` и т.д.              |
| **Math**               | `Math.floor`, `Math.random` и т.д.                               |
| **Встроенные функции** | `alert`, `prompt`, `eval` (специфичны для платформы)             |

## Как BorisType обрабатывает ограничения

| Ограничение                     | Решение BorisType                                          |
| ------------------------------- | ---------------------------------------------------------- |
| Нет `let`/`const`               | Транспиляция в `var` с hoisting                            |
| Нет `for...of`                  | Транспиляция в `for...in` (работает на массивах)           |
| Нет стрелочных функций          | Транспиляция в обычные функции                             |
| Нет шаблонных литералов         | Транспиляция в конкатенацию строк (`+`)                    |
| Нет деструктуризации            | Транспиляция в отдельные присваивания                      |
| Нет spread                      | Транспиляция в `concat` / `Object.assign` polyfills        |
| Нет замыканий (захват var)      | Генерация цепочки `__env`                                  |
| Нарушенный приоритет операторов | Добавление явных скобок                                    |
| Ternary внутри бинарных         | Извлечение во временные переменные (expression extraction) |
| Нет безопасности свойств        | Обертка в `bt.getProperty()`                               |
| `$` в именах переменных         | Генерация префикса `__dollar__`                            |

## Лучшие практики

### 1. Избегайте полагаться на приоритет операторов

```typescript
// ❌ Рискованно
const result = a + b * c;

// ✅ Явно
const result = a + b * c;
```

### 2. Не используйте объекты как Map со сложными ключами

```typescript
// ❌ Плохо
const map: Record<object, string> = {};

// ✅ Хорошо
const map: Record<string, string> = {};
```

### 3. Будьте осторожны с замыканиями

```typescript
// ❌ Не сработает как ожидается без транспиляции
for (var i = 0; i < 3; i++) {
  callbacks.push(() => console.log(i)); // Все выведут 3
}

// ✅ Позвольте BorisType обработать это (генерирует __env)
for (let i = 0; i < 3; i++) {
  callbacks.push(() => console.log(i)); // Корректно выведет 0, 1, 2
}
```

### 4. Тщательно тестируйте

Поведение runtime BorisScript может отличаться от JavaScript. Всегда тестируйте в целевом окружении.
