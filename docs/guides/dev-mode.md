# Dev Mode (Watch & Auto-Deploy)

## Обзор

Dev mode (`btc dev`) предоставляет:

- **Watch mode** - пересборка при изменении файлов
- **Инкрементальная линковка** - только изменённые файлы
- **Авто-push** - автоматический деплой на WebSoft HCM

## Базовое использование

```bash
npx btc dev
```

## Координация мультипакетных проектов

Для монорепозиториев с несколькими пакетами:

1. **Начальная сборка** - ждёт завершения ВСЕХ пакетов
2. **Полная линковка** - линкует все пакеты вместе
3. **Watch** - инкрементальные пересборки + линковки
4. **Авто-push** - после каждой успешной линковки

Пример `btconfig.json`:

```json
{
  "packages": {
    "backend": "./backend",
    "frontend": "./frontend"
  }
}
```

Dev mode следит за обоими пакетами и координирует сборку.

## Инкрементальные сборки

После начальной сборки:

- Перекомпилируются только изменённые файлы
- Копируются в dist/ только изменённые файлы
- node_modules копируются только при начальной сборке

**Результат:** ~10x более быстрые пересборки

## Авто-push

Включён по умолчанию. Отключить:

```bash
npx btc dev --no-push
```

**Debounce:** задержка 500мс после последнего изменения

**Соединение:** Persistent-сессия (переиспользует WshcmClient)

## Фильтрация пакетов

Отслеживаются только пакеты с полем `ws:package`.

**Исключаются:**

- Библиотечные пакеты (`type: "library"`) (временная мера, будет исправлено в будущем)
- Пакеты без `ws:package`

## Жизненный цикл

```
Start
  ├─> Начальная сборка (все пакеты)
  ├─> Ожидание завершения всех сборок
  ├─> Полная линковка
  ├─> Начальный push
  └─> Watch-цикл:
       ├─> Обнаружено изменение файла
       ├─> Инкрементальная пересборка
       ├─> Инкрементальная линковка (только изменённые файлы)
       ├─> Push
       └─> Reload модулей
```

## Устранение проблем

### Некоторые пакеты не пересобираются

- Проверьте поле `ws:package` в package.json
- Убедитесь что пакет указан в btconfig.json

### Изменения не отправляются

- Проверьте сетевое соединение
- Проверьте credentials в btconfig.properties
- Проверьте логи сервера

### Начальная сборка зависла

- В одном из пакетов могут быть ошибки компиляции
- Проверьте `btc build` для каждого пакета отдельно
