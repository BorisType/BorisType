# Примеры BorisType

Коллекция примеров использования BorisType компилятора и системы линковки.

## Примеры

### 📦 [single/](single/) - Legacy режим (одиночный пакет)

Демонстрирует классический подход к линковке с использованием настроек в `package.json`.

**Когда использовать:**
- Простые проекты с одним основным пакетом
- Миграция с существующих проектов
- Не нужна сложная структура

**Ключевые особенности:**
- Конфигурация через `ws:package` и `ws:root` в package.json
- Автоматическое добавление системных зависимостей (polyfill)
- Поддержка npm зависимостей

**Быстрый старт:**
```bash
cd single
npm install
npm run build
npm run link
```

---

### 🎯 [multi/](multi/) - Multi-package режим

Демонстрирует новый подход к линковке множественных пакетов через `btconfig.json`.

**Когда использовать:**
- Проекты с несколькими компонентами (backend, frontend, shared)
- Микросервисная архитектура
- Нужна гибкая настройка путей линковки

**Ключевые особенности:**
- Конфигурация через `btconfig.json`
- Shorthand нотация для BT пакетов
- Поддержка обычных директорий со статическими файлами
- Автоматическое добавление системных зависимостей (polyfill)

**Быстрый старт:**
```bash
cd multi
npm install
npm run link
```

---

## Сравнение режимов

| Характеристика | Single (Legacy) | Multi-package |
|----------------|-----------------|---------------|
| **Конфигурация** | `package.json` | `btconfig.json` |
| **Файл конфигурации** | ws:package, ws:root | linking.packages[] |
| **Количество пакетов** | 1 основной + зависимости | Любое количество |
| **Системные зависимости** | ✅ Автоматически | ✅ Автоматически |
| **Shorthand нотация** | ❌ | ✅ Для BT пакетов |
| **Типы пакетов** | app, lib | BT пакеты, обычные директории |
| **Приоритет** | Низкий | **Высокий** |
| **Использование** | Простые проекты | Сложные проекты |

## Приоритет конфигурации

Если в проекте присутствует `btconfig.json` с секцией `linking`, система автоматически использует **multi-package режим**, игнорируя настройки `ws:package` и `ws:root` в корневом `package.json`.

```
btconfig.json (linking) > package.json (ws:package, ws:root)
```

## Структура dist/ после линковки

### Single режим
```
dist/
├── wt/
│   ├── myapp/           # ws:root из package.json
│   │   ├── index.js
│   │   └── node_modules/
│   └── bt/
│       ├── filemap/     # Автоматически
│       └── polyfill/    # Автоматически
└── source/
    └── api_ext.xml
```

### Multi режим
```
dist/
├── wt/
│   ├── bt/
│   │   ├── polyfill/    # Автоматически (зависимость компилятора)
│   │   └── filemap/     # Автоматически
│   ├── backend/         # target из ws:root (BT пакет)
│   │   ├── index.js
│   │   └── node_modules/
│   └── frontend/        # target из btconfig.json (обычная директория)
│       └── index.html
└── source/
    └── api_ext.xml
```
